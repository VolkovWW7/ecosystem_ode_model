# Программный комплекс математического моделирования и калибровки процессов биоокисления (BioOx Simulation)

Данное программное обеспечение предназначено для моделирования динамики микробиологических систем в биореакторе. В основе комплекса лежит жесткая система обыкновенных дифференциальных уравнений (ОДУ), описывающая взаимодействие популяций автотрофов (водорослей *Chlorella*) и гетеротрофов (бактерий), а также динамику субстратов, детрита, БЖУ и энергетических пулов (АТФ).

Программа оснащена кроссплатформенным графическим интерфейсом и модулем автоматического подбора (калибровки) кинетических параметров на основе генетических алгоритмов.

## Основные возможности

* **Численное интегрирование систем ОДУ:** Использование современного адаптивного метода `LSODA` (из библиотеки `SciPy`) для решения жестких биологических систем.
* **Интерактивный GUI:** Удобный интерфейс на базе **PySide6 (Qt)**, позволяющий на лету изменять начальные условия, параметры среды и константы.
* **Встроенная научная визуализация:** Графики популяций, детрита, БЖУ и АТФ интегрированы непосредственно в окна вкладок интерфейса благодаря связке `Matplotlib` и Qt.
* **Двухступенчатая гибридная калибровка:** Уникальный алгоритм автоматического подбора параметров, сочетающий глобальный эволюционный поиск (первый этап) и прецизионное локальное уточнение в суженных границах с математической полировкой (второй этап). Это предотвращает застревание алгоритма на «плато» локальных минимумов.
* **Мультиметрический контроль сходимости:** Система оценивает качество кинетики одновременно по трем статистическим критериям: абсолютная (RMSE), относительная (RMSRE) и нормированная по размаху (NRMSE) среднеквадратичные ошибки. Оптимизация ведется адаптивно под каждую метрику на разных этапах.
* **Многопоточность:** Тяжелые вычисления калибровки вынесены в отдельный фоновый поток (`QThread`), что сохраняет интерфейс программы отзывчивым во время расчетов.
* **Управление конфигурациями:** Импорт и экспорт настроек и результатов калибровки в универсальном формате JSON.

## Архитектура проекта

* `main.py` — Главная точка входа, реализующая паттерн MVC (контроллер), управляющая потоками данных и жизненным циклом приложения.
* `gui.py` — Модуль графического интерфейса. Описывает структуру главного окна, логику виджетов, вкладок и Canvas для графиков.
* `mathmodel.py` — Математическое ядро модели. Содержит систему ОДУ, дефолтные параметры, функции расчета и генерации графиков.
* `calibrate_gui.py` — `calibrate_gui.py` — Модуль многоступенчатой оптимизации. Реализует расчёт векторов ошибок, фильтрацию нестабильных решений ОДУ, динамическое сужение поисковых границ и последовательный запуск эволюционных эпох.

## Методология и логика калибровки

Поскольку биологические системы обладают высокой нелинейностью, оптимизация параметров по одной лишь относительной ошибке (RMSRE) часто приводит к застреванию алгоритма из-за сильного завышения штрафов на малых концентрациях (в лаг-фазе). 

Для решения этой проблемы в комплексе реализован **двухступенчатый калибровочный конвейер**:

1. **Ступень 1: Глобальная макро-подгонка (Целевая метрика — NRMSE)** Используется алгоритм дифференциальной эволюции (`differential_evolution`) в широких биологических границах параметров. Задача этапа — вывести расчетные кривые модели в правильный масштаб и поймать общий тренд эксперимента, игнорируя локальные шумы малых величин.
   
2. **Ступень 2: Локальная микро-подгонка (Целевая метрика — RMSRE)** Программа автоматически берет лучший вектор параметров из Ступени 1 и динамически сужает границы поиска вокруг него (в пределах $\pm 20\%$). В этих жестких рамках запускается повторный поиск, ориентированный на минимизацию относительной ошибки, а на финальном шаге включается математическая полировка (метод локального спуска `polish=True`) для поиска абсолютного дна «зоны оптимума».

## Требования к системе и окружение

Для запуска исходного кода требуется **Python 3.10+**.

Основные зависимости:
* `PySide6` — Графический интерфейс Qt.
* `numpy` — Векторно-матричные вычисления.
* `scipy` — Решение ОДУ (`solve_ivp`) и оптимизация (`differential_evolution`).
* `matplotlib` — Построение графиков и холсты интеграции в Qt.

## Установка и запуск

1. Клонируйте репозиторий или перейдите в папку с проектом:
   ```bash
    git clone https://github.com/VolkovWW7/ecosystem_ode_model.git
    cd ecosystem_ode_model
   ```
2. Создайте виртуальное окружение
    ```bash
    # Windows
    python -m venv venv
    venv\Scripts\activate

    # Linux / macOS
    python3 -m venv venv
    source venv/bin/activate
    ```
3. Установите зависимости
    ```bash
    pip install pyside6 numpy scipy matplotlib
    ```
4. Запуск приложения
    ```bash
    python3 main.py
    ```
